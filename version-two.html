OpenTSDB 2.0 Roadmap

The OpenTSDB community has been growing and many developers have contributed patches, new features and ideas to the project. We are proud to start development on version 2.0 that will integrate many of these features in the official repository. 2.0 will maintain backwards compatibility with the previous release though there maybe some slight changes in the HTTP and Java APIs. Below is a roadmap for the next release:

Timeline

  • DONE: July 2013 - Initial Release Candidate GitHub
  • August 2013 - Release Candidate 2
  • October 2013 - Official Release

We'll do our best to hit these dates and can certainly use all of the help we can get.

Improvements

These are just some of the improvements on the planning board:

  • DONE: Improved (lock-less) UID assignment
  • DONE: Faster query speeds for complex or aggregated queries
  • DONE: More aggregators with interpolation bypass
  • WIP: New website and documentation including a user guide

New Features

Some of the new features we're planning to integrate include:
  • DONE: A configuration file for storing properties
  • DONE: Full RESTful HTTP API that provides access to all of OpenTSDB's features including storing data points via HTTP
  • DONE: Pluggable data formatters that can accept or emit data in various formats. The default will be JSON though implementations could be created for XML, ProtoBuf or others
  • DONE: Annotations that allow tracking of events associated with a particular timeseries or global notes tied to a timestamp
  • DONE: Metadata for timeseries, metrics, tag names and tag values so you can find out what is stored in OpenTSDB and add your own notes
  • DONE: Metadata trees that organize timeseries into a directory like structure
  • DONE: Pluggable search abstraction layer to publish metadata and annotations to your search engine of choice, such as Elastic Search
  • DONE: Millisecond timestamp support, enabled as an option for users who need greater resolution
  • DONE: Pluggable real-time publisher to push incoming data out to systems such as RabbitMQ or Memcache
  • Debian (DONE), CentOS and FreeBSD packages with init scripts

Requests

Have a feature you would like to see in OpenTSDB? Check the Issues list to see if someone is working on it or already posted a request. Also post a notice to the Users Group.

Features for 2.0 are essentially locked in at this time so that we can finish the code and focus on bug hunting. Features we'll work on for 2.1 or later include:

  • Increment call to add or subtract from a specified data point
  • Query language
  • Top N queries
  • Rollups
  • Last value queries
  • Pluggable data abstraction layer that will allow the use of data stores other than HBase. Initially we'll release a Cassandra plugin
  • More graphing options

Documentation

As work progresses on 2.0, we'll update the documentation here. If you would like to contribute to the docs, fork the opentsdb.net project on github and edit the Sphinx source files, compile the HTML and issue a pull request. We're also looking for anyhone who has design chops to help make the website and new documentation pretty, so if you're interested, please contact us.

Contributions

We need all of the help we can get so post your thoughts to the users group, then read the Development guide and Contributing page. Create a fork in GitHub and publish your changes to the "next" branch. Issue a pull request and we'll try and fit it in.
